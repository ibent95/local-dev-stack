# 001 · LDS Custom Apps — Planning

**Date:** 2026-07-06
**Status:** Complete (planning finalized)
**Type:** Planning

---

## Overview

This development cycle introduces three custom LDS applications — **Analytics**,
**Tasks**, and **Wiki** — each consisting of a Hono-based API backend and a
modern frontend UI. Additionally, **Vaultwarden** (self-hosted Bitwarden
compatible password manager) is integrated as a service. The goal is to provide
internal tools for data analytics, task management, and knowledge sharing, all
running within the local-dev-stack infrastructure.

---

## 1 · Architecture

Each app follows the LDS pattern:

```
┌─────────────┐     ┌─────────────┐
│   UI (FE)   │────▶│   API (BE)  │────▶ Database
│  Port 417x  │     │  Port 300x  │     (SQLite / Postgres)
└─────────────┘     └─────────────┘
```

- **API**: Hono on Node.js with Drizzle ORM, served via `@hono/node-server`
- **UI**: Framework-specific SPA/SSR, served by its own dev server
- **Network**: Each app uses an external `lds-network` for service discovery
- **Profiles**: Each app has its own compose profile (off by default)

## 2 · Apps

### 2.1 Analytics

**Purpose:** Real-time analytics dashboard with charts, stat cards, and data visualization.

**Stack:**
- **API**: Hono + Drizzle ORM + SQLite
- **UI**: Nuxt 4 (Vue 3) + Tailwind CSS v4 + Chart.js (vue-chartjs)

**Features:**
- Dashboard with stat cards (total views, visitors, etc.)
- Line chart for traffic over time
- Bar chart for top pages
- Pie chart for traffic sources
- Dark theme with responsive layout

**Key Dependencies:**
- `nuxt@^4.4.8`, `vue@^3.5.38`
- `chart.js@^4.4.0`, `vue-chartjs@^5.3.0`
- `@nuxtjs/tailwindcss@^6.14.0`, `tailwindcss@^4.0.0`
- `lucide-vue-next` (icons)

### 2.2 Tasks

**Purpose:** Kanban-style task management with drag-and-drop.

**Stack:**
- **API**: Hono + Drizzle ORM + SQLite
- **UI**: Angular 22 + Angular Material + CDK drag-drop

**Features:**
- Kanban board with columns (To Do, In Progress, Done)
- Drag-and-drop task reordering
- Task creation, editing, deletion
- Angular Material UI components
- Dark theme

**Key Dependencies:**
- `@angular/core@^22.0.0`, `@angular/material@^22.0.0`
- `@angular/cdk@^22.0.0` (drag-drop)
- `typescript@~6.0.0`

### 2.3 Wiki

**Purpose:** Knowledge base and documentation dashboard.

**Stack:**
- **API**: Hono + Drizzle ORM + SQLite
- **UI**: Next.js 16 (React 19) + Tailwind CSS v4

**Features:**
- Document listing and viewing
- Search functionality
- Dark theme with responsive layout
- API proxy via Next.js rewrites

**Key Dependencies:**
- `next@^16.2.10`, `react@^19.2.0`
- `tailwindcss@^4.0.0`, `@tailwindcss/postcss@^4.0.0`
- `lucide-react` (icons)

### 2.4 Vaultwarden

**Purpose:** Self-hosted Bitwarden-compatible password manager.

**Stack:**
- **Server**: Vaultwarden (Rust) Docker image
- **UI**: Built-in Bitwarden web vault

**Features:**
- Password vault management
- Browser extension support
- Self-hosted, no external dependencies
- Admin panel for user management

## 3 · API Design

All three APIs share a common Hono-based architecture:

```
src/
├── index.ts          # Entry point, CORS, routes
├── db.ts             # Drizzle ORM client
├── schema.ts         # Table definitions
├── routes/
│   ├── sites.ts      # /api/sites
│   ├── tasks.ts      # /api/tasks
│   └── docs.ts       # /api/docs
└── types/
    └── api.ts        # Shared TypeScript types
```

**Common patterns:**
- `@hono/node-server` for Node.js HTTP serving
- Drizzle ORM with SQLite (portable, zero-config)
- `verbatimModuleSyntax` with `.js` extensions
- `NodeNext` module system
- Health check at `GET /api/health`

## 4 · Docker Configuration

### 4.1 Dockerfiles

Each UI and API has its own `Dockerfile` using the shared `lds/node-dev` base:

```dockerfile
ARG NODE_VERSION=26.3
FROM lds/node-dev:${NODE_VERSION}
USER root
WORKDIR /app
COPY package.json package-lock.json* ./
RUN npm install
COPY . .
```

**Key decisions:**
- `--ignore-scripts` for Nuxt UI (prevents `nuxt prepare` before source copy)
- No `--legacy-peer-deps` (proper peer dep resolution)
- `npm rebuild` for native module support
- Files are `COPY`'d (not bind-mounted) — rebuild required for changes

### 4.2 Docker Compose

Each app has its own profile:

```yaml
analytics:
  analytics-api:
    profiles: [analytics]
    build: { context: configs/analytics/api }
  analytics-ui:
    profiles: [analytics]
    build: { context: configs/analytics/ui }

tasks:
  tasks-api:
    profiles: [tasks]
    build: { context: configs/tasks/api }
  tasks-ui:
    profiles: [tasks]
    build: { context: configs/tasks/ui }

wiki:
  wiki-api:
    profiles: [wiki]
    build: { context: configs/wiki/api }
  wiki-ui:
    profiles: [wiki]
    build: { context: configs/wiki/ui }
```

## 6 · Build & Run

```bash
# Build all app images
docker compose --profile analytics --profile tasks --profile wiki build

# Start specific apps
./lds.sh up analytics        # Analytics only
./lds.sh up tasks wiki       # Tasks + Wiki

# Start everything
./lds.sh up all
```
